Chapter 1 \'b7 Theory

Backpropagation & Computational Graphs

\

How neural networks compute gradients efficiently using the chain rule. This is exam material \'97 understand every step.

\
\ \

What is a Computational Graph?

\

A computational graph breaks a complex function into a sequence of simple operations (nodes). Each node knows its local gradient (how its output changes w.r.t. its inputs). Backpropagation flows gradients backwards through the graph using the chain rule.

\ \
\
\

Two Node Types

\
\
\ + (Addition)\ Gradient distributor: passes upstream gradient unchanged to both inputs\
\
\ \'d7 (Multiply)\ Gradient switcher: each input gets the other input's value \'d7 upstream gradient\
\
\
\
\

Chain Rule (Core)

\
\ $$\\frac\{\\partial \\text\{Loss\}\}\{\\partial x\} = \\frac\{\\partial \\text\{Loss\}\}\{\\partial z\} \\cdot \\frac\{\\partial z\}\{\\partial x\}$$\
\

Each node multiplies the incoming upstream gradient by its local gradient.

\
\
\ \

Worked Example: f(x,y,z) = (x+y)\'b7z

\

This is exactly the example from lecture slides 55\'9658 by Prof. Godinho de Matos.

\ \
\ Function decomposition:
\ Step 1 \'97 Let $q = x + y$ (addition node)
\ Step 2 \'97 Let $f = q \\cdot z$ (multiplication node)
\ Values: $x = -2,\\; y = 5,\\; z = -4$\
\ \

Step 1: Forward Pass

\

Compute values left \uc0\u8594 right through the graph:

\ \
\ \ \
\ \
\ $$q = x + y = -2 + 5 = 3 \\qquad f = q \\cdot z = 3 \\times (-4) = -12$$\
\ \

Step 2: Local Gradients (Partial Derivatives)

\

Before running the backward pass, compute the local gradient at each node:

\ \
\
Local Gradients \'97 Compute These First
\
\
\ \uc0\u8706 f/\u8706 z = q\ = $3$  (because $f = q \\cdot z$, so derivative w.r.t. $z$ is the other factor $q$)\
\
\ \uc0\u8706 f/\u8706 q = z\ = $-4$  (because $f = q \\cdot z$, so derivative w.r.t. $q$ is the other factor $z$)\
\
\ \uc0\u8706 q/\u8706 x = 1\ = $1$  (because $q = x + y$, adding 1 to $x$ adds 1 to $q$)\
\
\ \uc0\u8706 q/\u8706 y = 1\ = $1$  (same reason)\
\
\
\ \

Step 3: Backward Pass (Chain Rule)

\

Start from the output (upstream gradient = 1) and work right \uc0\u8594 left, multiplying upstream gradient \'d7 local gradient at each step:

\ \
\
\
1
\
\
At the output: $\\partial f / \\partial f = 1$ (always 1)
\

This is the starting upstream gradient. The loss gradient w.r.t. f is assumed to be 1.

\
\
\
\
2
\
\
Gradient for z:
\
$$\\frac\{\\partial f\}\{\\partial z\} = \\underbrace\{\\frac\{\\partial f\}\{\\partial z\}\}_\{\\text\{local\}\} \\times \\underbrace\{1\}_\{\\text\{upstream\}\} = q \\times 1 = 3 \\times 1 = \\boxed\{3\}$$
\
\
\
\
3
\
\
Gradient for q (flows into the + node):
\
$$\\frac\{\\partial f\}\{\\partial q\} = \\underbrace\{\\frac\{\\partial f\}\{\\partial q\}\}_\{\\text\{local\}\} \\times \\underbrace\{1\}_\{\\text\{upstream\}\} = z \\times 1 = -4 \\times 1 = \\boxed\{-4\}$$
\
\
\
\
4
\
\
Gradient for x (chain rule through + node):
\
$$\\frac\{\\partial f\}\{\\partial x\} = \\underbrace\{\\frac\{\\partial f\}\{\\partial q\}\}_\{\\text\{upstream\}\} \\times \\underbrace\{\\frac\{\\partial q\}\{\\partial x\}\}_\{\\text\{local\}\} = (-4) \\times 1 = \\boxed\{-4\}$$
\
\
\
\
5
\
\
Gradient for y (same logic):
\
$$\\frac\{\\partial f\}\{\\partial y\} = \\underbrace\{\\frac\{\\partial f\}\{\\partial q\}\}_\{\\text\{upstream\}\} \\times \\underbrace\{\\frac\{\\partial q\}\{\\partial y\}\}_\{\\text\{local\}\} = (-4) \\times 1 = \\boxed\{-4\}$$
\
\

    \

Why This Matters for Neural Networks

\

In a real neural network, every weight is like $x$, $y$, or $z$ in this example. The loss function is like $f$. Backpropagation computes $\\partial\\text\{Loss\}/\\partial w$ for every single weight in one efficient backward pass. The weight update rule is then:

\ \
\ $$w_\{\\text\{new\}\} = w_\{\\text\{old\}\} - \\lambda \\cdot \\frac\{\\partial \\text\{Loss\}\}\{\\partial w\}$$\
\ \

For the digit recognition network with 13,002 parameters, backpropagation computes all 13,002 gradients in one pass \'97 far more efficient than computing each separately.

\
\ Architecture: z = w\uc0\u8321 x\u8321 + w\u8322 x\u8322 + b  \u8594   \'e2 = \u963 (z)  \u8594   L = (\'e2 \u8722 y)\'b2

\
\
\uc0\u8706 L/\u8706 \'e2 (loss)
2(\'e2\u8722 y) = \u8722 0.620
\
\uc0\u8706 \'e2/\u8706 z (sigmoid)
\'e2(1\u8722 \'e2) \u8776 0.214
\
\uc0\u8706 z/\u8706 w\u8321

\
\uc0\u8706 z/\u8706 w\u8322
x\u8322 = 1
\
\uc0\u8706 z/\u8706 b

\

Chapter 3 \'b7 Theory

Batch Size, Steps & Epochs

\

Understanding how dataset size, batch size, and epochs interact to determine training dynamics. Prof. Godinho de Matos said this will be tested.

\
\ \

Core Definitions

\ \
\
\ N\ Dataset size \'97 total number of training examples in your dataset\
\
\ Batch Size (B)\ Number of examples per gradient update \'97 how many samples are processed before weights are updated once\
\
\ Step / Iteration\ One weight update \'97 process one batch \uc0\u8594 compute loss \u8594 backprop \u8594 update weights\
\
\ Epoch\ One full pass through the entire dataset \'97 all N samples have been seen once\
\
\ Steps/Epoch\ Number of weight updates per epoch \'97 equals \uc0\u8968 N / B\u8969 (ceiling division)\
\
\ \

The Key Formula

\ \
\ $$\\text\{Steps per epoch\} = \\left\\lceil \\frac\{N\}\{B\} \\right\\rceil \\qquad \\text\{Total updates\} = \\text\{Epochs\} \\times \\text\{Steps per epoch\}$$\
\ \
\ \uc0\u8968 \u8969 = Ceiling: Always round up. If $N=850$ and $B=100$, you get $\\lceil 8.5 \\rceil = 9$ steps. The last batch has only $850 - 8 \\times 100 = 50$ samples, but it still counts as a step.\
\ \

Effect of Changing Batch Size

\ \
\
\

\uc0\u8593 Larger Batch Size

\
    \
  • Fewer steps per epoch
    • \
  • More accurate gradient estimate
    • \
  • More memory required (GPU)
    • \
  • Fewer total weight updates
    • \
  • May generalize worse (converges to sharp minima)
    • \
  • Faster wall-clock time per epoch
    • \
\
\
\

\uc0\u8595 Smaller Batch Size

\
    \
  • More steps per epoch
    • \
  • Noisier gradient estimate
    • \
  • Less memory required
    • \
  • More total weight updates
    • \
  • Often generalizes better (implicit regularization)
    • \
  • Slower but can escape local minima
    • \
\

\ \

$$\\left\\lceil \\frac\{1000\}\{100\} \\right\\rceil = 10 \\text\{ steps/epoch\} \\qquad 10 \\times 20 = \\boxed\{}
